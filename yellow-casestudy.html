Case Study: Transforming Yellow – Digitizing Succession Planning in India

Introduction

Yellow set out to tackle one of India’s most pressing yet overlooked problems: succession planning. With billions of dollars lying unclaimed and countless families navigating a system riddled with inefficiencies and corruption, Yellow aimed to provide a seamless, digital-first solution for users to create a living will. However, before my arrival as CTO, the startup was struggling—burning money on an external software agency that failed to deliver a robust product.

In just 6–8 months, I built an in-house engineering team, inherited a subpar codebase, re-architected the entire tech stack, and successfully launched a Flutter-based mobile app along with a fully functional web app. This case study details how we executed this turnaround.

The Challenge: A Dysfunctional Development Process

Before I joined, Yellow relied on an outsourced software agency that failed to deliver a scalable, maintainable product. Key challenges included:

  • Poor Code Quality: The Flutter app had been built with an inconsistent architecture, leading to frequent crashes and performance bottlenecks.
  • Backend Scalability Issues: The Node.js backend was monolithic, poorly structured, and lacked proper security and efficiency optimizations.
  • Frontend Limitations: The React-based web app was fragmented, with a lack of reusable components and proper state management.
  • High Burn, Low Output: Despite significant investments, the agency's deliverables were slow, unoptimized, and required constant rework.

The Turnaround: Building an In-House Team & Re-Architecting the Stack

Step 1: Assembling a Lean Engineering Team

I hired a small team of young, highly motivated engineers who were eager to build something meaningful. By bringing development in-house, we reduced dependencies on third-party vendors and gained full control over the technology.

Step 2: Rebuilding the Flutter App

One of the first tasks was to overhaul the Flutter app. The previous implementation had fragmented state management, redundant widgets, and inefficient API handling.

Key Improvements:

  • Adopted a Clean Architecture: Implemented a layered architecture following Flutter's best practices (Presentation, Domain, Data layers) to improve scalability.
  • State Management Overhaul: Moved to Riverpod for efficient state management, reducing unnecessary rebuilds and improving performance.
  • Optimized API Calls: Implemented caching mechanisms and batch API requests to reduce latency.
  • Security Enhancements: Integrated secure authentication and encrypted local storage to protect sensitive user data.

Step 3: Revamping the Node.js Backend

The inherited Node.js backend was inefficient, making data retrieval slow and resource-intensive. We restructured it into a modular, scalable system.

Key Improvements:

  • Microservices Architecture: Broke the monolithic backend into independent, scalable services.
  • Database Optimization: Implemented indexing and query optimization for faster retrieval.
  • Security Enhancements: Integrated OAuth, JWT authentication, and rate limiting to safeguard user data.
  • Automated Deployment: Introduced CI/CD pipelines for faster and more reliable releases.

Step 4: Refining the React Web App

The web platform had a disorganized codebase, making it hard to maintain and extend. We revamped it with a focus on performance and user experience.

Key Improvements:

  • Component-Driven Architecture: Refactored UI into reusable React components for consistency and maintainability.
  • Optimized State Management: Switched from prop-drilling to Redux Toolkit for more predictable state handling.
  • Responsive Design: Ensured a seamless experience across devices using TailwindCSS and Material UI.
  • Accessibility & Performance: Implemented lazy loading, server-side rendering, and Lighthouse audits to improve loading speed and SEO.

The Result: A Successful Launch in Record Time

Thanks to the architectural improvements and efficient execution, we successfully launched a robust mobile app and web platform within 6–8 months. The impact was immediate:

  • Significant Cost Savings: Eliminating reliance on the external agency reduced operational costs drastically.
  • Improved Performance: App crash rates dropped by over 70%, and API response times improved by 50%.
  • User Adoption: Post-launch, we saw a steady increase in user registrations and positive feedback from early adopters.
  • Scalability Achieved: The platform was now built to handle future growth without major rework.

Conclusion

By taking over Yellow’s technical leadership, I turned around a struggling development process, built a high-performing in-house team, and delivered a market-ready solution in under eight months. The experience reinforced the importance of hands-on leadership, clean architecture, and rapid execution in scaling an early-stage startup. Yellow is now well-positioned to disrupt succession planning in India with a digital-first approach.
